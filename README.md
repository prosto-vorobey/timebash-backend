# Timebash – Personal Activity Time Tracker

> [!NOTE]
> The source code and xml-documentation are in English. The web interface and this README with a comprehensive description are provided in Russian.

**Timebash** — это трекер пользовательской активности в течение дня. Пользователь фиксирует свои занятия и их продолжительность. Система агрегирует данные и предоставляет статистику по категориям через API.

## Цели проекта
- Создание законченного MVP для портфолио и личного использования
- Демонстрация навыков построения веб-приложений на ASP.NET Core с использованием Entity Framework Core
- Применение современных практик: контейнеризация инфраструктурных сервисов (PostgreSQL, Seq), логирование (Serilog), CI/CD через GitHub Actions
- Создание расширяемой архитектуры с разделением на backend и клиентские части

## Ключевые возможности (MVP)
Бэкенд предоставляет REST API со следующими возможностями:
- **Аутентификация и авторизация** — регистрация, вход, выдача JWT‑токенов. Доступ к ресурсам контролируется на уровне контроллеров.
- **Управление журналами и активностями** — CRUD-операции для журналов (`Journal`) и активностей (`Activity`) с указанием времени начала и завершения.
- **Система категорий** — создание категорий, назначение их активностям с поддержкой связи "многие ко многим".
- **Статистика** — эндпоинты для подсчёта времени по категориям, журналам и пользователю, включая активности без категорий.
- **Обработка ошибок и валидация** — собственный middleware перехватывает исключения, возвращая соответствующие HTTP-статусы и тела ошибок. Входные данные валидируются с помощью `FluentValidation`, ошибки валидации возвращаются в виде `ValidationErrorResponse`.
- **Rate Limiting** — ограничение частоты запросов через `SlidingWindowRateLimiter` для защиты от брутфорс-атак.
- **Документирование** — интерактивный Swagger UI (временно открыт для демонстрации) с поддержкой JWT-аутентификации и встроенная XML-документация контроллеров, сервисов и DTO.

## Архитектура
Проект разделён на две независимые части, что позволяет в будущем добавлять клиенты под другие платформы (Android, Desktop и т.п.):

### Backend (`Timebash.Backend`)
Проект разделён на слои с чётким разделением ответственности:
- **Timebash.API** — слой контроллеров, DI-контейнер, middleware
- **Timebash.Application** — бизнес-логика, сервисы, валидаторы, DTO-мапперы
- **Timebash.Core** — доменные сущности, интерфейсы репозиториев и сервисов, DTO, исключения
- **Timebash.Infrastructure** — реализация репозиториев, `DbContext`, миграции, Unit of Work
- **Timebash.Tests.Unit** — модульные тесты Application и Core
- **Timebash.Tests.Integration** — интеграционные тесты (в процессе наполнения)

Доменные сущности: `User`, `Journal`, `Activity`, `Category`, `ActivityCategory`, `UserSettings`.

### Web (`Timebash.Web`)
Razor Pages-приложение, взаимодействующее с Backend через HTTP API. Использует HTMX для динамического обновления страниц без полноценного SPA.

## Развёртывание и инфраструктура
Проект развёрнут на VPS (Ubuntu 24.04) с использованием следующего стека:

- **Reverse proxy** — Nginx маршрутизирует запросы на бэкенд и веб-приложение, а также обслуживает SSL-сертификаты. Сертификат для demo-части получен через Certbot.
- **Управление процессами** — API и веб-часть запущены как systemd-сервисы, что обеспечивает автоматический перезапуск и интеграцию с системой.
- **Изоляция деплоя** — развёртывание выполняется от отдельного пользователя с минимально необходимыми правами.
- **CI/CD** — GitHub Actions автоматизирует:
  - *pull request* — запуск модульных тестов и проверку сборки;
  - *push тега версии* — деплой на сервер с автоматическим перезапуском сервисов.
- **Инфраструктурные сервисы** — PostgreSQL и Seq запущены в Docker-контейнерах через Docker Compose.
- **Миграции** — при старте API автоматически применяются миграции Entity Framework Core (`MigrateAsync`), поддерживая схему БД в актуальном состоянии.
